# Handoff — python-vis-sort-web

## What this is
A Python sorting algorithm visualizer that runs in the browser. It's a portfolio project ported from Tkinter to Pygame + Pygbag (WASM). Lives at `https://sethrobinson29.github.io/python-vis-sort-web/`.

## State of the project
- All code is written and working locally (desktop pygame + pygbag local server tested)
- Deployed to GitHub Pages via GitHub Actions on every push to `main`
- Custom loading screen (`template.tmpl`) is in place — dark theme, app title, auto-start

## How to run locally
```bash
pip install pygame-ce pygbag

# Desktop
python main.py

# Browser — open http://localhost:8000
python -m pygbag --template template.tmpl main.py
```

## Key files
| File | Purpose |
|------|---------|
| `main.py` | Entry point. Pygame init, `Button`/`Slider` classes, async game loop |
| `sorter.py` | `Sorter` class. All sorts are `async def`. Draw throttled to `numBars//100` |
| `template.tmpl` | Custom Pygbag HTML loading screen. Edit CSS/HTML below `# BEGIN BLOCK` |
| `.github/workflows/deploy.yml` | CI: installs deps, runs `pygbag --build --template template.tmpl main.py`, deploys `build/web` |

## Things to know
- **Adding a sort**: add `async def newSort(self)` to `Sorter`, add a button in `main.py`, handle the action string in the event loop
- **Template changes**: edit `template.tmpl`. The Python block (lines 1–103) controls loading logic. The HTML/CSS block starts after `# BEGIN BLOCK`. Rebuild with pygbag to see changes.
- **Pygbag gotchas**: no `sys.exit()`, no `time.sleep()`, no `SysFont`, all blocking operations must be async
- **`build/` is gitignored** — regenerated by pygbag on each build
